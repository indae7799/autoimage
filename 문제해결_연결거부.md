# ERR_CONNECTION_REFUSED 해결 방법

## 문제 원인

브라우저가 서버가 완전히 시작되기 전에 열렸을 때 발생합니다.

---

## 해결 방법

### 방법 1: 수동으로 서버 실행 (가장 확실함)

**터미널 1 (백엔드):**
```cmd
cd C:\Users\jungindae\Downloads\상세페이지\backend
pip install -r requirements.txt
uvicorn main:app --reload
```

**터미널 2 (프론트엔드):**
```cmd
cd C:\Users\jungindae\Downloads\상세페이지\frontend
npm install
npm run dev
```

**터미널 2에서 다음 메시지가 나올 때까지 기다리세요:**
```
➜  Local:   http://localhost:5173/
```

그 다음 브라우저에서 http://localhost:5173 접속

---

### 방법 2: 실행.bat 수정 버전 사용

수정된 `실행.bat` 파일이 서버가 준비될 때까지 더 오래 기다립니다.

1. `실행.bat` 다시 실행
2. 서버 창이 열리면 메시지를 확인하세요
3. "웹사이트: http://localhost:5173" 메시지가 나올 때까지 기다리세요
4. 브라우저가 자동으로 열립니다

---

### 방법 3: 서버 상태 확인

**백엔드 확인:**
- http://localhost:8000/docs 접속 시도
- API 문서가 보이면 백엔드는 정상 작동 중

**프론트엔드 확인:**
- 작업 표시줄에서 "프론트엔드" 창 확인
- "Local: http://localhost:5173" 메시지가 보이는지 확인

---

## 체크리스트

- [ ] 프론트엔드 서버 창이 열려 있나요?
- [ ] 각 창에 에러 메시지가 있나요?
- [ ] 포트 5173이 다른 프로그램에서 사용 중인가요?

---

## 일반적인 문제

### "포트가 이미 사용 중입니다"
- 다른 프로그램이 포트를 사용 중
- 작업 관리자에서 Python이나 Node 프로세스 종료
- 컴퓨터 재시작

### "모듈을 찾을 수 없습니다"
- 패키지 설치 필요:
  ```cmd
  cd backend
  pip install -r requirements.txt
  
  cd ..\frontend
  npm install
  ```

### 서버 창이 바로 닫힘
- 에러 메시지 확인
- Python/Node.js 설치 확인
- .env 파일 확인

---

## 추천 방법

**가장 확실한 방법은 수동 실행입니다!**

두 개의 명령 프롬프트 창을 열고 각각에서 서버를 실행하면 정확히 어떤 문제가 있는지 확인할 수 있습니다.
